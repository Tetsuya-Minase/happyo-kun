# タスク完了時のチェックリスト

## コード品質

### TypeScriptチェック
このプロジェクトには明示的なlint/formatコマンドが定義されていませんが、TypeScriptのstrictモードが有効なため、以下を確認してください：

```bash
# ビルドを実行してTypeScriptエラーがないことを確認
pnpm build
```

ビルドが成功すれば、TypeScriptの型エラーはありません。

### コードスタイル確認
- 既存のコードスタイルに従っているか確認
- 命名規則が一貫しているか確認
- コメントが適切に記述されているか確認

## 機能テスト

### ローカル開発環境での確認
```bash
# 開発サーバーを起動
pnpm dev

# 各アプリケーションをブラウザで確認
# - start: http://localhost:3000
# - slide: http://localhost:3030
```

### ビルド確認
```bash
# プロダクションビルドを実行
pnpm build

# ビルド結果をプレビュー
pnpm preview
```

### 確認項目
- [ ] すべてのスライドが正しく表示される
- [ ] API呼び出しが正常に動作する
- [ ] エラーハンドリングが適切に機能する
- [ ] レスポンシブデザインが動作する (モバイル表示確認)
- [ ] 各プレゼンテーションへのルーティングが正しい

## 新しいプレゼンテーションを追加した場合

### 必須確認事項
- [ ] `apps/slide/slides/【ディレクトリ名】/slides.md` が作成されている
- [ ] `pnpm build` が成功する
- [ ] `apps/slide/dist/【ディレクトリ名】/` にビルド結果が生成されている
- [ ] `apps/start/dist/【ディレクトリ名】/` に統合ビルドがコピーされている
- [ ] `/【ディレクトリ名】/1` でアクセスできる

## Git コミット前

### 変更内容の確認
```bash
# ステータス確認
git status

# 差分確認
git diff

# ステージングされた変更の確認
git diff --staged
```

### コミットメッセージ規約
- `feat:` 新機能の追加
- `fix:` バグ修正
- `docs:` ドキュメントの変更
- `style:` コードスタイルの変更 (フォーマット等)
- `refactor:` リファクタリング
- `test:` テストの追加・修正
- `chore:` その他の変更 (ビルド設定等)

例：
```bash
git commit -m "feat: Add new presentation about TypeScript"
git commit -m "fix: Fix API error handling in hello endpoint"
```

## デプロイ前

### 最終確認
- [ ] すべてのビルドが成功する
- [ ] ローカルでの動作確認が完了している
- [ ] 不要なコンソールログやデバッグコードが削除されている
- [ ] 環境変数が適切に設定されている (必要な場合)
- [ ] CLAUDE.md が最新の状態を反映している (大きな変更がある場合)

## プルリクエスト作成時

### 確認事項
- [ ] わかりやすいタイトルと説明を記載
- [ ] 変更内容のスクリーンショットを添付 (UI変更の場合)
- [ ] 関連するIssueをリンク (存在する場合)
- [ ] GitHub Actions のチェックがパスしている

## 注意事項
- このプロジェクトには自動テストが設定されていないため、手動での動作確認が重要です
- Cloudflare Pagesへのデプロイは、GitHub Actionsを通じて自動的に行われます
- _redirects ファイルは使用しません（Cloudflare Pagesは自動的にSPAルーティングを処理します）
